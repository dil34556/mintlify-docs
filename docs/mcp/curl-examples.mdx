# cURL Examples for Telminator MCP

Learn how to interact with Telminator's MCP server using cURL commands with practical examples.

## Prerequisites

- cURL installed on your system
- Access to Telminator MCP server endpoint
- Basic understanding of JSON-RPC protocol

## Server Endpoint
```
http://mcp.telminator.com:8888/mcp
```

## Basic Operations

### 1. List Available Tools

Discover all tools available on the MCP server:
```bash
curl -X POST 'http://mcp.telminator.com:8888/mcp' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/list",
    "params": {}
  }'
```

### 2. Call a Tool

Execute a specific tool with parameters:
```bash
curl -X POST 'http://mcp.telminator.com:8888/mcp' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -d '{
    "jsonrpc": "2.0",
    "id": 3,
    "method": "tools/call",
    "params": {
      "name": "list_a2a_mcp_server",
      "arguments": {
        "query": "test"
      }
    }
  }'
```

## JSON-RPC Protocol

The Telminator MCP server uses JSON-RPC 2.0 protocol with event-stream responses.

### Request Format
```json
{
  "jsonrpc": "2.0",           // Protocol version (required)
  "id": 1,                     // Request ID (required)
  "method": "method_name",     // Method to call (required)
  "params": {                  // Parameters (optional)
    "key": "value"
  }
}
```

### Response Format

The server returns responses in Server-Sent Events (SSE) format:

**Success Response:**
```
event: message 
data: {
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    // Result data
  },
  "isError": false
}
```

**Error Response:**
```
event: message
data: {
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32601,
    "message": "Method not found"
  },
  "isError": true
}
```

## Available Methods

| Method | Description | Parameters |
|--------|-------------|------------|
| `tools/list` | List all available tools | None |
| `tools/call` | Execute a specific tool | `name` (string), `arguments` (object) |

## Tool Details

### list_a2a_mcp_server

Searches for an agent and MCP server configurations based on any user input query.

**Input Schema:**
```json
{
  "type": "object",
  "properties": {
    "query": {
      "type": "string",
      "description": "Natural language query describing the task or server type needed"
    }
  },
  "required": ["query"]
}
```

**Output:**
- Returns agent details and relevant server URLs with configurations
- Includes `a2a_result` (agent-to-agent results)
- Includes `search_result` with URLs and server configurations

## Error Codes

| Code | Meaning | Description |
|------|---------|-------------|
| `-32700` | Parse error | Invalid JSON was received |
| `-32600` | Invalid request | The JSON sent is not a valid request |
| `-32601` | Method not found | The method does not exist |
| `-32602` | Invalid params | Invalid method parameters |
| `-32603` | Internal error | Internal JSON-RPC error |

## Tips & Best Practices

<AccordionGroup>
  <Accordion title="Pretty Print JSON Output">
    Use `jq` to format JSON responses (note: SSE format requires extracting data first):
```bash
curl -X POST 'http://mcp.telminator.com:8888/mcp' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -d '{
    "jsonrpc":"2.0",
    "id":2,
    "method":"tools/list",
    "params":{}
  }' | grep "^data: " | sed 's/^data: //' | jq '.'
```
  </Accordion>

  <Accordion title="Save Response to File">

## Understanding the Response Structure

The `list_a2a_mcp_server` tool returns a structured response containing:

### Response Fields

- **content**: Array of text content containing the JSON result
- **structuredContent**: Parsed structured data with:
  - **a2a_result**: Agent-to-agent communication results (may be null)
  - **search_result**: Search results containing:
    - **urls**: Array of relevant server URLs
    - **server_configs**: Array of server configuration objects

### Server Configuration Object

Each server config includes:
- **name**: Name of the MCP server
- **url**: GitHub or repository URL
- **description**: Description of the server's capabilities
- **server_config**: JSON string containing the MCP server configuration

## Troubleshooting

<Steps>
  <Step title="Verify Server is Running">
    Test basic connectivity:
```bash
curl -I http://mcp.telminator.com:8888/mcp
```
  </Step>
  
  <Step title="Check JSON Syntax">
    Validate your JSON before sending:
```bash
echo '{
  "jsonrpc":"2.0",
  "id":1,
  "method":"tools/list",
  "params":{}
}' | jq '.'
```
  </Step>
  
  <Step title="Verify Headers">
    Always include both Content-Type and Accept headers:
    - Content-Type: application/json
    - Accept: application/json, text/event-stream
  </Step>
  
  <Step title="Review Error Messages">
    Check the error code and message in the response for debugging. Look for `"isError": true` in the response.
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Missing Accept Header">
    **Issue:** Response format is incorrect or incomplete.
    
    **Solution:** Always include the Accept header:
```bash
-H 'Accept: application/json, text/event-stream'
```
  </Accordion>

  <Accordion title="Query Returns No Results">
    **Issue:** The search query doesn't match any servers.
    
    **Solutions:**
    - Try broader search terms
    - Use task-based queries like "send email" instead of technical terms
    - Check spelling in your query
  </Accordion>

  <Accordion title="Parsing SSE Format">
    **Issue:** Response contains `event: message` and `data:` prefixes.
    
    **Solution:** This is the Server-Sent Events format. Extract the data portion:
```bash
grep "^data: " | sed 's/^data: //'
```
  </Accordion>
</AccordionGroup>
